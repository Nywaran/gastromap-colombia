CRISP-ML Methodology

Cross-Industry Standard Process for Machine Learning

What is CRISP-ML?

CRISP-ML is an evolution of CRISP-DM (Cross-Industry Standard Process for Data Mining), specifically adapted for modern Machine Learning projects. It establishes iterative, well-defined phases oriented toward business results.

Unlike its predecessor, CRISP-ML emphasizes model integration into real systems and post-deployment monitoring — fundamental aspects in the lifecycle of current AI solutions. (Schmid, Wurst & Wirth, 2020)

Phase 1 — Business & Data Understanding

Business Objectives

Translate business needs into concrete machine learning project goals.

Key Questions

Identify the specific questions the model must answer.

Data Assessment

Explore data quality, relevance, and sufficiency.

Feasibility Analysis

Consider limitations and risks, ensuring technical and organizational viability.

View our Business Understanding →

Phase 2 — Data Engineering

Selection

Identify relevant data sources.

Cleaning

Correct errors and handle missing values.

Transformation

Create new features and normalize data.

Documentation

Record all transformations performed.

View our Data Engineering →

Phase 3 — Model Engineering

Algorithm Selection

Choose appropriate techniques based on problem type and data characteristics.

Training

Apply prepared data to the selected algorithm.

Hyperparameter Tuning

Optimize configurations for improved performance.

Validation

Verify performance with data not used during training.

Phase 4 — Model Evaluation

Performance Metrics

Select and apply appropriate indicators to measure model effectiveness.

Cross-Validation

Evaluate the model's generalization capacity with different data subsets.

Risk Analysis

Identify potential issues such as overfitting, underfitting, or biases.

Expert Review

Validate results with domain specialists for practical relevance.

Phase 5 — Deployment

Production Implementation

Integrate the validated model into the real operating environment.

Documentation & Training

Create technical and user manuals along with training sessions.

Protocol Establishment

Define procedures for continuous monitoring and incident management.

View our Deployed Dashboard →

Phase 6 — Monitoring & Maintenance

Continuous Supervision

Regular monitoring of model performance in production.

Drift Detection

Early identification of performance deterioration due to data changes.

Model Updating

Periodic retraining with new data to maintain effectiveness.

Risk Review

Continuous vulnerability assessment and quality control adaptation.
