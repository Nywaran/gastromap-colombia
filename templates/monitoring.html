Phase 6

Monitoring & Maintenance

Continuous supervision, drift detection, model updating, and risk review

6.1 Continuous Supervision

Data Source Monitoring

The RNT dataset on datos.gov.co is updated periodically by MinCIT. The system should re-download and re-process the RNT at least quarterly to capture new restaurant registrations, closures, and category changes. The last update used in this version was May 16, 2026.

Ranking Stability Checks

After each data refresh, compare the new top 20 ranking against the previous version. If more than 5 municipalities change positions by more than 10 places, investigate whether the shift reflects real changes or data quality issues.

API Health Monitoring

Monitor the Flask application's response times and availability on Render.com. The /api/municipalities endpoint should respond in under 500ms. Set up uptime monitoring (e.g., UptimeRobot) to detect outages.

User Feedback Collection

Track which filters and regions users interact with most on the dashboard. High engagement with specific regions may indicate demand for more granular data in those areas.

6.2 Drift Detection

Data drift occurs when the statistical properties of input data change over time, potentially degrading model performance. For this system, key drift indicators include:

Drift TypeWhat to MonitorThresholdAction
Volume Drift Total RNT records per refresh >10% change between quarters Investigate if new categories were added or if data pipeline changed
Category Drift Number of gastronomy subcategories New subcategories appearing Update regex filter if new gastronomy-related categories are introduced
Distribution Drift GTPI score distribution shape Classification counts change >20% Re-evaluate bin edges or normalization parameters
Geographic Drift New municipalities in DIVIPOLA Any new codes appearing Add to geographic base and re-run climate estimation

6.3 Model Updating Strategy

Q

Quarterly Data Refresh

Download the latest RNT from datos.gov.co. Re-run python data_processor.py to regenerate processed_dataset.json, eda_report.json, and engineering_report.json. Push to GitHub → Render auto-deploys.

A

Annual Weight Review

Once per year, review GTPI weights with tourism domain experts. Consider implementing AHP (Analytic Hierarchy Process) with pairwise comparisons to derive data-informed weights. Document any weight changes and their justification.

E

Event-Driven Updates

If DANE releases updated DIVIPOLA codes (e.g., new municipalities created), or if IDEAM makes historical climate data available via API, trigger an immediate pipeline update to incorporate the improved data.

6.4 Risk Review

Data Source Discontinuation

If datos.gov.co changes the RNT API endpoint or format, the data pipeline will break. Mitigation: keep a local backup of the last successful download and monitor the API status in the quarterly refresh script.

Medium Risk

Platform Migration

Render.com's free tier may change terms or impose limitations. The application is platform-agnostic (standard Flask + Gunicorn) and can be deployed to Railway, Fly.io, or any cloud provider with minimal configuration changes.

Low Risk

Regulatory Changes

If MinCIT modifies the RNT category taxonomy (adding, renaming, or merging gastronomy categories), the regex filter GASTRO|BAR(?!RAN) may need updating to capture new categories or exclude false positives.

Medium Risk

Security

The application is read-only (no user input stored, no database, no authentication). Attack surface is minimal. HTTPS is provided by Render.com. Dependencies should be updated periodically for security patches.

Low Risk

6.5 Future Improvements Roadmap

Real Climate Data

Integrate WorldClim bioclimatic variables or IDEAM weather station data via API to replace the coordinate-based estimation model with actual measurements.

Demand-Side Metrics

Incorporate Google Trends data, TripAdvisor reviews, or DANE tourism survey results to add a demand dimension to the GTPI formula.

Supervised Model

Use current GTPI classifications as pseudo-labels to train a supervised model (Gradient Boosting, Random Forest) that can predict potential from raw features without the full pipeline.

Temporal Weighting

Assign higher weights to recently registered establishments (2024-2026) vs older ones (2019-2021) to capture growth momentum and identify emerging destinations.
