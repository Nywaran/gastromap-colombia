5. Normalization and Scaling

Min-Max Normalization

x_normalized = (x - x_min) / (x_max - x_min)

All five GTPI components are scaled to the [0, 1] range using min-max normalization. This ensures that features with different original scales (counts ranging 0-3,854 vs. index values ranging 0-1) contribute proportionally to the weighted composite score.

Log Transformation Justification

Three features (gastro_count, total_providers, gastro_employees) exhibit extreme right-skew: Bogotá has 3,854 gastronomy establishments while the median municipality has only 5. Without log transformation, min-max normalization would compress 99% of municipalities into the [0, 0.03] range, making the index meaningless for differentiating non-capital cities. The log(1+x) transformation compresses the range from [0, 3854] to [0, 8.26], producing a near-normal distribution that allows meaningful discrimination across all tiers.

6. Scoring Model — GTPI

GTPI = 0.35 × GD + 0.20 × GV + 0.20 × TI + 0.15 × CC + 0.10 × GE
35%

Gastronomy Density (GD)

Log-normalized count of gastronomy establishments. Highest weight — directly measures the core variable.

20%

Gastronomy Variety (GV)

Number of distinct subcategories. More diversity indicates a richer culinary offering.

20%

Tourism Infrastructure (TI)

Log-normalized total tourism providers. Tourists need hotels and transport, not just restaurants.

15%

Climate Comfort (CC)

Comfort index (0-1). Optimal range: 18-28°C, moderate rainfall (~1500 mm/year).

10%

Gastronomy Employment (GE)

Log-normalized employee count. Indicates sector maturity and economic scale.
